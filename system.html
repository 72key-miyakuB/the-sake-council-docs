売上管理（キャッシュレス統合）

自動集計→可視化→報告

キャッシュレス店舗の強みを活かし、日次・週次の売上報告を自動生成。施策判断に繋げる。

自動化ピックアップ：売上集計・レポート自動生成 NOW KPI

何を自動化する？（To-Be）

  • 決済データの自動取り込み（日次）
  • 売上の日次/週次レポート自動生成（PDF/HTML）
  • 客単価・曜日別・時間帯別の可視化

どうシステム化する？（例）

  • 決済/POSのCSV/APIを定期取得 → sales_raw に保存
  • 集計バッチ（Python/Apps Script）→ sales_daily / sales_weekly
  • テンプレHTMLに差し込み → 自動で「日報」生成
出力：日報HTML / 週報PDF
入力処理出力運用者
POS/決済データ（CSV/API） 自動取込→自動集計→グラフ/指標生成 日報（売上/客単価/前年差） 店長（確認のみ）
KPI例：売上（週）／客単価／客数／曜日別偏り／前年差・前週比
アラート：異常値検知（売上急落・客単価急変） SOON RISK

仕組み

  • 前週平均との差が -20% など閾値超えで通知
  • 通知先：Slack/LINE/メール（店長＋オーナー）
例：if sales_today < avg_last_4weeks*0.8 then notify

コスト管理（原価・経費・労務）

自動収集→原価率/人件費率→改善

既存の原価・経費アプリ基盤を活用し、「毎日見える」「超えたら知らせる」運用へ。

自動化ピックアップ：原価・経費の自動取り込み／原価率算出 NOW KPI

何を自動化する？

  • レシート/請求の取り込み（画像→データ化）
  • 品目ごとの単価マスター連携 → 原価自動計算
  • 原価率（週/月）を自動算出

どうシステム化する？

  • 仕入れ入力：カメラ/OCR → purchase_raw（既存アプリ）
  • 単価マスター：master_unit_price を参照して自動換算
  • 売上（sales）と突合 → cogs_rate 生成
設定：目標原価率 __%（許容レンジも合意）
追加：予算・許容レンジ超過のアラート SOONRISK
  • 原価率が許容上限 __% 超過 → 店長に通知
  • 人件費率が許容上限 __% 超過 → シフト最適化提案
労務：勤怠→人件費率の自動更新（シフト連動） SOONKPI

仕組み

  • 勤怠（打刻）を日次で取得 → 時給マスターと掛け合わせ
  • 売上データと突合して人件費率を自動表示
  • 翌週の必要人数を推定（売上予測ベース）

在庫・発注・廃棄

棚卸→発注→欠品防止

在庫は「入力を最小化」し、発注提案（おすすめ発注）まで自動化する。

自動化ピックアップ：発注提案（在庫・回転・予約から自動） SOONKPI

何を自動化する？

  • 棚卸入力（スマホ）→在庫の自動更新
  • 消費ペースから「発注候補」を自動算出
  • 予約（人数）から必要量を上乗せ
  • 欠品リスクの通知

どうシステム化する？

  • 在庫テーブル：inventory（SKU/数量/期限）
  • 売上・提供数から消費推定：usage_est
  • 発注案：po_suggest を生成し、発注書CSVに出力
発注先別にCSVを分ける（業者ごと）
期限・廃棄：期限アラート／廃棄ログ自動集計 SOONRISK
  • 入荷時に期限入力（or ロット入力）→ 期限 3日前 に通知
  • 廃棄入力を簡略化（理由プリセット）→ 月次ロスを自動集計
  • ロス上位品目を自動ランキング → 発注基準見直しへ
日本酒在庫：ボトル管理（開栓日／提供杯数／在庫見える化） LATERKPI
  • 開栓時に「開栓日」を登録 → 提供杯数から残量推定
  • 劣化リスク（開栓後日数）をアラート
  • おすすめ（回転が落ちている酒）を自動で提案
※日本酒QRシステムと連動できると強い（商品ID共通化）。

メニュー管理（表示・更新・QR）

一括更新→即反映

メニューは「更新が面倒」だと劣化するので、更新を1箇所に集約して自動反映。

自動化ピックアップ：マスタ編集→メニューHTML/QR自動生成 NOWKPI

何を自動化する？

  • 商品マスター更新（価格/在庫/おすすめ）
  • タブレット/スマホ向けメニューを自動生成
  • 品切れの自動非表示（or グレー表示）

どうシステム化する？

  • マスター：menu_master（CSV/Sheet）
  • ビルド：テンプレに差し込み → menu.html 生成
  • QR：menu.html のURLをQR化（更新不要）
推奨：タブレット＋お客様スマホの併用（回遊性UP）
原価連動：原価率から価格の妥当性チェック（提案） LATERKPI
  • 原価（単価マスター）と販売価格から「原価率」を自動算出
  • 原価率が高すぎる商品をハイライト（値付け/量の検討）

シフト管理（自動化）

希望→自動割当→公平性

テスト段階の基盤を「運用に耐える」形へ。欠勤時のバックアップまで含めて設計。

自動化ピックアップ：希望収集→自動シフト生成（公平性制御） NOWKPI

何を自動化する？

  • 希望/NGのフォーム入力 → 自動集計
  • 必要人数（曜日×時間帯）に合わせて自動割当
  • 稼働率の偏りを自動検知→均等化提案
  • 欠勤時の代替候補を自動リストアップ

どうシステム化する？

  • 入力：Google Form/アプリ → requests
  • 最適化：制約（NG/連勤/公平性）でスコアリング
  • 出力：shift.csv / カレンダー表示 / スタッフ通知
公平性KPI：月間稼働時間の標準偏差を可視化
勤怠連携：打刻→給与概算→人件費率まで自動化 SOONKPI
  • 勤怠データを取り込み、時給・深夜・残業を自動計算
  • 売上と突合して「日次の人件費率」を自動表示

顧客の声（満足度・改善）

収集→集計→改善

アンケートとスタッフメモを集約し、改善タスクに自動変換する。

自動化ピックアップ：アンケート自動集計＋改善タスク化 SOONKPI

何を自動化する？

  • QRアンケート（3問）→自動集計
  • 自由記述をカテゴリ分類（味/接客/雰囲気）
  • 改善案を「タスク」へ自動起票

どうシステム化する？

  • Google Form → スプレッドシート
  • 分類：Apps Script or AI分類 → feedback_tags
  • タスク化：Trello/Notion/Sheet（既存タスク管理へ）

イベント・プロモ企画（スケジュール化）

年間→月→週→日

企画を属人化させず、自動で「今週やること」に落ちる仕組み。

自動化ピックアップ：年間企画カレンダー→週次ToDo自動生成 SOONKPI

何を自動化する？

  • 年間イベント表（季節/祝日）を登録
  • 毎週「今週の企画ToDo」を自動生成
  • SNS投稿の下書き（テンプレ）を自動生成

どうシステム化する？

  • 年間計画：promo_calendar（Sheet）
  • 週次バッチ：今週対象を抽出 → weekly_todo
  • テンプレ：投稿文/画像案を生成（AI活用）
効果測定：投稿→来店/予約の相関を簡易に記録（UTM/予約メモ）
SNS運用：投稿計画→投稿素材管理→効果モニタリング LATERKPI
  • 投稿カレンダー（週次）を自動生成
  • 素材フォルダ命名規則を統一（探す時間をゼロへ）
  • 数値（保存/リーチ/来店）を毎週自動でサマリー
